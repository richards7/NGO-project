## Overview
**Arogya Camp OS** (Health Camp Hub) is an offline-first medical camp management application designed for NGOs. It facilitates digitized workflows for rural health camps—including patient registration, capturing vitals, queuing, doctor consultations, prescriptions, pharmacy dispensing, inventory management, and analytics reporting.

The system is built on **TanStack Start** (Vite + React 19 + Nitro server engine) using TypeScript, with UI styling provided by **Tailwind CSS v4** and components built using **Radix UI** primitives via a **Shadcn** setup.

## Data Flow

1. **Authentication Gate:** The user arrives at `/` (`LoginPage`), selects a role, and logs in. The session is stored in `localStorage` under `arogya.session`.
2. **Layout Ingestion:** If authenticated, the user is navigated to the home page associated with their role. The route resolves to `_app.tsx`, which wraps the page output within `AppShell`.
3. **Information Routing:** TanStack Start handles route resolution and matches files within `src/routes/` to the page rendering outlets.
4. **Offline Synchronization:** The `useOnlineStatus` hook simulates offline status and mocks auto-sync timing for pending requests.

## Conventions

- **Naming:** React components are camelCase or PascalCase (e.g., `app-shell.tsx`, `use-mobile.tsx`); routes follow file-system routing patterns defined by TanStack Start (`_app.patients.new.tsx`).
- **Structure:** Modularized separation between routing code (`src/routes/`), application context utilities (`src/lib/`), and layout primitives (`src/components/`).
- **Styling:** CSS variables defined in standard stylesheet coupled with Tailwind CSS utility classes and modern animations.

## Technical Debt

- **Mock Data Dependence:** The app currently relies heavily on static mock datasets defined in `demo-data.ts`. It does not hit real database APIs or REST servers.
- **Offline Persistence Level:** The offline support is simulated and does not utilize a true IndexedDB/ServiceWorker system to cache network requests, though it provides the UI and hook structure for it.
